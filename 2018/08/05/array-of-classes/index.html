什么是类数组，类数组是如何定义的？

发表于 2018-08-05 | 分类于 javascript | 热度 °C
字数统计 1,418

类数组，在刚开始学习前端的时候就有很多人已经接触了这种东西，但一直也没有深究到底是如何去定义的。今天在学习的时候偶然注意到了这个字眼，突然回想起像jQuery或vue这样的框架都有用上这种设计，那么浏览器是怎么看待这件事的呢？

就先拿经典的jQuery作测试吧。在一个加载有jQuery的页面上打开控制台，随便获取一个元素节点:

我们可以看到在 chrome 浏览器上它是以一个(类)数组的形式展示出来，但其实我们都知道，它只是一个对象。那么浏览器是如何识别出来的呢？

首先来猜想下，引擎可能是根据某种属性是否存在来判断是否为“数组”，毕竟JavaScript的数组本质上也是一个对象。
为了验证这个猜想，我在控制台输入Array()，然后根据这个原生数组的__proto__属性，来查看数组的原生方法。同时创建一个新对象，用来“模拟”数组的方法。

emmmm，果然变成了个“数组”，虽说前面还提示着是一个Object。这也证明着这些属性并非要在原型上才管用。没关系，接着继续改变constructor属性，但我们不能直接修改arr原型上的constructor，因为这会影响到全局对象的constructor。所以我们这里需要做一个调整，改为函数的形式~

唔…验证了最初的设想，名字也都改了。紧着缩小范围，逐步定位是哪些属性在“作祟”，最后的出的结论是：

  1. length属性是必须要有的，毕竟这是数组的基本特性。而且要对象里有这么个属性才行，光是原型上有是不行的。
  2. 其次我发现只要少了splice这个属性，引擎就直接判断你可能是个“假”数组

随后又想到了HTMLCollection和NodeList是否也是这样的，经过一番尝试后并没有真正的完全模拟成功，也有可能是内置的问题~

再接着就想测一测是否全部浏览器都是这种展现形式，答案是：并不是统一的，经过测试，主流的浏览器只有 V8 引擎（chrome、Safari）是以数组的方式展现，高冷的 Firefox 浏览器打印出来的结果是:
Object { length: 0 }

好吧，折腾了半天得出了这个结果好像有点惨，其实冷静想一想来也正常，毕竟这种东西一般也不会有什么规范，会规定要如何展现会比较好，各凭厂商喜好。就像以前文章提过的console的实现方式也是如此。

好啦~ 正儿八经的水了半篇之后，我们话归正题。

类数组对象(Array-link Object)，一个看似像数组类型的对象。一般我将其分为两类，一类是在和DOM打交道的时候，调用获取DOM列表所返回的NodeList或HTMLCollection，再或者是函数内的Arguments对象。它们的共同点就是：都拥有length属性，同时还有看似像下标的数值属性。

它们并没有真正的数组方法，因为它们并没有继承数组的方法：

但也有办法让它如同正常的数组一样工作，通过调用call来给目标方法指定上下文~

或许到这里就会有人问了，怎么这类数组这么麻烦的？就连基础的数组方法都没有？？ 诶~且慢，这里就不得不提另一种“野生”的类数组方法啦~
最“野生”的类数组方法可能只有个length属性，因为这是类数组最基础的定义。

再来换个角度想，数组的那些方法其实都是语言留给我们的语法糖，本质上数组也只是个特殊的对象。同为对象，我们自己也可以围绕着length手动模拟这些方法或新增一些方法。

开头所提到的JQuery，就是一个典型的例子。jQuery获取的节点列表就是一个类数组，它的forEach函数也早在ES5实现之前，就添加入库，而jQuery也是因简洁高效而闻名。

这个例子就能告诉我们不要因为没有数组便捷的语法糖就唾弃它… js是一个弱类型语言，同时代表它是一个灵活的语言~因此需要我们灵活的使用~
