关于小站

关于内容

在日常工作中，我们总遇到不同的问题和难点。或许我们这一次遇到了这种业务场景，几个月后或者更久又碰到了相同的场景，虽然我们有了前面的经验知道大致思路该如何处理，但是时隔已久再重新回想的话需要一定的时间，中间还可能丢失一些细节。但此时如果有文字纪录的话就能很快的回想起当初的场景，这也是笔者开始写博客的契机。

写博客说到底就是梳理知识的一个过程，将碎片化的知识通过写作转换为较为完整的知识结构和解决方案，这样一来可以温故知新，二来还可以回馈社区。

这个博客主要分享工作中所遇到的一些疑难杂症的解决方案、具体业务场景的分析与实现、梳理过后的 web 相关知识点、计算机基础相关的学习笔记等。内容更多偏向实际应用与现成的解决方案。

关于博客

博客是使用 Hexo 框架所搭建，采用 Next 主题并通过 github pages 部署的。因为多次有朋友问我博客怎么搭建，干脆又写一篇介绍如何从零搭建博客的文章。

博客主要是用于分享学习成果的，内容大多都是也是通过梳理自己的知识点后才发布出来。在学习过程中自然会有通过其他博主分享出来的技术文章中收益到东西，因此有借鉴之处都会在底部标注参考资料的来源。

博客内文章遵循 BY-NC-SA 协议，允许转载但需署名和注明出处。

博客内文章一般也不会主动去转载文章，如需转载也必定会提前询问作者的授权。文章内也可能会引用网络图片做配图(主要是早期的文章)，如有图片版权相关的问题，请联系笔者替换或删除。

关于笔者

笔者是一个现役社畜，对数据处理、数据可视化相关的前端工作会比较熟悉，也了解一些服务端的知识。

主要会在博客和 Front-End-Lab 这两块地方佛系写作，前者的更新内容已经在上文说明~ 后者则是一系列更细化的 web 技术体系细节，未来理想情况会抽离为一个知识索引(实际上是一系列的 flag)。

如果在阅读中遇到了什么疑惑或者是有误的，也可以跟笔者交流与反馈~
